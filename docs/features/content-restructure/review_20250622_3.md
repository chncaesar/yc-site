## Issues

### Minor
- [m1] **Identical CSS blocks duplicated across 4 files** — `src/pages/articles/index.astro:55-72`, `src/pages/interview/index.astro:50-66`, `src/pages/health/index.astro:50-66`, `src/pages/events/index.astro:50-66`. For SEO best practices, the `<title>` should mirror the page's `<h1>`. (The brief specifies these exact values, so this is a spec-level choice — flagging for awareness only.)
- [m3] **No empty-state handling when a collection has zero posts** — All four pages render an empty `<ul class="post-list">` with no items if the queried collection is empty (e.g., events currently has only 1 post; if it were removed, the page would show a blank list with header/footer). Consider adding a fallback message like "暂无文章".
